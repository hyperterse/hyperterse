Step-by-step techniques for diagnosing and resolving issues.

## Enable debug logging

First step: increase log verbosity.

```bash
# Debug level
hyperterse run -f config.terse --log-level 4

# Or use verbose flag
hyperterse run -f config.terse -v
```

**Log Levels:**

| Level | Value | Shows |
|-------|-------|-------|
| ERROR | 1 | Only errors |
| WARN | 2 | Warnings and errors |
| INFO | 3 | General info (default) |
| DEBUG | 4 | Everything |

## Check generated documentation

### OpenAPI spec

Verify endpoints are generated correctly:

```bash
curl http://localhost:8080/docs | jq
```

Check:
- All expected endpoints present
- Input schemas correct
- Descriptions accurate

### LLM documentation

```bash
curl http://localhost:8080/llms.txt
```

## Test queries directly

### Using curl

```bash
curl -X POST http://localhost:8080/query/get-user \
  -H "Content-Type: application/json" \
  -d '{"userId": 123}' \
  -v  # verbose output
```

### Check response

```json
{
  "success": true,
  "error": "",
  "results": [...]
}
```

If `success: false`, check the `error` field.

## Test database connection

### PostgreSQL

```bash
psql "${DATABASE_URL}" -c "SELECT 1"
```

### MySQL

```bash
mysql -h host -u user -p -e "SELECT 1"
```

### Redis

```bash
redis-cli -u "${REDIS_URL}" PING
```

## Validate configuration

Before running, validate your config:

```bash
# Check YAML syntax
python -c "import yaml; yaml.safe_load(open('config.terse'))"

# Or use yq
yq '.' config.terse
```

## Common debug scenarios

### Query returns empty results

1. Test query directly in database
2. Check input values match data
3. Verify template substitution
4. Look for WHERE clause issues

### Query fails silently

1. Enable DEBUG logging
2. Check database server logs
3. Test query in database directly
4. Look for permission issues

### Slow queries

1. Check database EXPLAIN plan
2. Add indexes if needed
3. Review result set size
4. Check connection pool usage

### Hot reload not working

1. Verify using `dev` command
2. Check file watch path
3. Look for syntax errors in config
4. Check file permissions

## MCP debugging

### List tools

```bash
curl -X POST http://localhost:8080/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "method": "tools/list", "id": 1}'
```

### Test tool call

```bash
curl -X POST http://localhost:8080/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {"name": "get-user", "arguments": {"userId": "1"}},
    "id": 1
  }'
```

Check `isError` field in response.

## Collecting debug information

When reporting issues, include:

1. **Hyperterse version**: `hyperterse --version`
2. **Configuration** (sanitize credentials)
3. **Full error message**
4. **Logs with DEBUG level**
5. **Steps to reproduce**

## Health check endpoint

Use `/docs` as a health check:

```bash
curl -f http://localhost:8080/docs > /dev/null && echo "OK"
```

## Resource monitoring

Monitor for resource issues:

```bash
# Cpu and memory
top -p $(pgrep hyperterse)

# Open connections
netstat -an | grep 8080 | wc -l

# File descriptors
ls /proc/$(pgrep hyperterse)/fd | wc -l
```
