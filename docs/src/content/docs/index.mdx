# Connect your data to your agents

Hyperterse is a high-performance runtime server that transforms database queries into **REST endpoints** and **MCP tools**. Define queries in a configuration file—Hyperterse handles execution, validation, documentation, and AI integration.

## How it works

Define your queries once in a simple configuration file:

```yaml title="config.terse"
adapters:
  production_db:
    connector: postgres
    connection_string: 'postgresql://user:pass@host:5432/db'

queries:
  get-user:
    use: production_db
    description: 'Retrieve a user by email address'
    statement: |
      SELECT id, name, email, created_at
      FROM users
      WHERE email = {{ inputs.email }}
    inputs:
      email:
        type: string
        description: 'User email address'
```

Run the server:

```bash
hyperterse run -f config.terse
```

Call your query via REST or MCP:

```bash
curl -X POST http://localhost:8080/query/get-user \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

## What's included

| Feature               | Description                             |
| --------------------- | --------------------------------------- |
| **REST Endpoints**    | Each query becomes `POST /query/{name}` |
| **MCP Protocol**      | JSON-RPC 2.0 tools at `POST /mcp`       |
| **OpenAPI Docs**      | Auto-generated at `GET /docs`           |
| **LLM Documentation** | AI-friendly docs at `GET /llms.txt`     |
| **Input Validation**  | Strong typing and validation by default |
| **Hot Reload**        | Instant feedback during development     |
